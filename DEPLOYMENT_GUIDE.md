# TryNex Lifestyle — Production Deployment Guide

## Architecture
- **Frontend**: Cloudflare Pages (free) — static SPA
- **Backend + DB**: Render.com (free tier) — Node.js API + PostgreSQL
- **Uptime**: UptimeRobot (free) — prevents Render free-tier sleep

---

## Step 1: Push to GitHub

Make sure all code is pushed to your GitHub repo:
- `georgelsmith333-hub/trynex-liestyle`

---

## Step 2: Deploy Backend on Render.com

1. Go to https://dashboard.render.com
2. Click **New** → **Blueprint**
3. Connect your GitHub repo (`georgelsmith333-hub/trynex-liestyle`)
4. Render reads `render.yaml` and auto-provisions:
   - **trynex-api** — Node.js web service (free tier)
   - **trynex-db** — PostgreSQL database (free tier)
5. After deployment, set these environment variables in Render dashboard:
   - `CORS_ORIGIN` → your Cloudflare Pages domain (e.g. `https://trynex-lifestyle.pages.dev`)
   - `ADMIN_PASSWORD` → already set to `Admins@Trynex` (change if desired)
   - `JWT_SECRET` → auto-generated by Render
6. Note your Render API URL: `https://trynex-api.onrender.com`

### What happens on first deploy:
- `pnpm install` installs dependencies
- Database tables auto-created via migrations
- 9 sample products auto-seeded
- Admin account auto-created

---

## Step 3: Deploy Frontend on Cloudflare Pages

1. Go to https://dash.cloudflare.com → **Pages**
2. Click **Create a project** → **Connect to Git**
3. Select your GitHub repo
4. Configure build settings:
   - **Build command**: `npm install -g pnpm && pnpm install && pnpm --filter @workspace/trynex-storefront run build`
   - **Build output directory**: `artifacts/trynex-storefront/dist/public`
   - **Root directory**: (leave empty — root of repo)
   - **Node.js version**: `20` (set in Environment Variables: `NODE_VERSION` = `20`)
5. Set environment variable:
   - `VITE_API_BASE_URL` = `https://trynex-api.onrender.com` (your Render URL from Step 2)
   - `NODE_ENV` = `production`
6. Deploy!

### After first deploy:
- Update `CORS_ORIGIN` on Render to match your Cloudflare Pages URL
- Redeploy Render if you change CORS_ORIGIN

---

## Step 4: Set Up UptimeRobot

1. Go to https://uptimerobot.com (free account)
2. Click **Add New Monitor**
3. Configure:
   - **Monitor Type**: HTTP(s)
   - **Friendly Name**: TryNex API
   - **URL**: `https://trynex-api.onrender.com/api/healthz`
   - **Monitoring Interval**: 5 minutes
4. Save — this pings your API every 5 minutes to prevent Render free-tier sleep

## Environment Variables Summary

### Render.com (Backend)
| Variable | Value | Notes |
|----------|-------|-------|
| `NODE_ENV` | `production` | Auto-set in render.yaml |
| `PORT` | `8080` | Auto-set in render.yaml |
| `DATABASE_URL` | (auto from Render DB) | Auto-linked in render.yaml |
| `JWT_SECRET` | (auto-generated) | Auto-generated in render.yaml |
| `ADMIN_PASSWORD` | (set in Render dashboard) | Required — set to `Admins@Trynex` or custom |
| `CORS_ORIGIN` | `https://your-site.pages.dev` | Set after Cloudflare deploy |

### Cloudflare Pages (Frontend)
| Variable | Value | Notes |
|----------|-------|-------|
| `VITE_API_BASE_URL` | `https://trynex-api.onrender.com` | Your Render API URL |
| `NODE_VERSION` | `20` | Required for build |
| `NODE_ENV` | `production` | Strips dev plugins |

## Admin Access

- **URL**: `https://your-site.pages.dev/admin/login`
- **Secret access**: Tap the ✦ symbol in the footer 5 times
- **Password**: Whatever you set as `ADMIN_PASSWORD` (default: `Admins@Trynex`)

## Health Check

- **Endpoint**: `GET /api/healthz`
- **Response**: `{"status":"ok"}`
- **Use for**: UptimeRobot, Render health checks

## Custom Domain (Optional)

### Cloudflare Pages:
1. Go to your Pages project → **Custom domains**
2. Add your domain (e.g. `trynexlifestyle.com`)
3. Follow DNS instructions

### Update CORS:
After adding a custom domain, update `CORS_ORIGIN` on Render to include it:
`https://trynexlifestyle.com,https://trynex-lifestyle.pages.dev`

## Troubleshooting

| Issue | Solution |
|-------|----------|
| API returns 503 | Render is sleeping — wait 30s or check UptimeRobot |
| CORS errors | Set `CORS_ORIGIN` on Render to your frontend URL |
| Admin login fails | Check `ADMIN_PASSWORD` env var on Render |
| Images not loading | Products use external URLs (Unsplash) — always available |
| Frontend shows blank | Check `VITE_API_BASE_URL` is set correctly on Cloudflare |
| Build fails on Cloudflare | Ensure `NODE_VERSION=20` is set in env vars |
