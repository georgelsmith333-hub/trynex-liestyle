#2 - Blogs, 3D Mockups & Promo Upgrades
What & Why
Three content and experience pillars to make TryNex feel like a premium flagship brand: 20 production-ready SEO-optimized blog articles seeded into the database, realistic photorealistic 3D garment mockups that replace any remaining SVG fallbacks, and visual/copy upgrades to the promotional offer system (spin wheel prizes, flash sale messaging, exit-intent UX).

Done looks like
20 published blog posts covering Bangladesh apparel, custom printing tips, fashion trends, gift ideas, corporate wear, and care guides — each with unique slug, keyword-rich title, full HTML body (500–900 words), excerpt, category, tags, cover image URL, author, and correct published_at dates spread over the last 6 months
Every blog post auto-generates valid BlogPosting + FAQPage JSON-LD (where applicable) via the existing SEOHead component
The 3D viewer shows photo-quality fabric textures on all five product types (t-shirt, hoodie, long-sleeve, cap, mug); no SVG placeholder ever appears for users with WebGL2 support; loading skeleton is polished
The garment GLB models are regenerated with higher-poly geometry and proper UV seams so photorealistic cotton/ceramic textures wrap convincingly
Spin wheel prizes updated: clearer prize copy, higher perceived value (e.g. "FREE Delivery", "৳100 Off", "10% Off", "Buy 1 Get 1 on Accessories"), wheel sectors visually distinct and branded
Flash sale bar: animated gradient, countdown urgency copy aligned with Bangladesh sale seasons (Eid, Puja, New Year)
Exit-intent popup: premium design, stronger value proposition headline, clear trust signals
Abandoned cart popup: shows item thumbnails, urgency timer, and a dynamically calculated savings figure
All promotional components have zero console errors and work on mobile viewport
Out of scope
Actual email delivery for forgot-password (stub only — covered in security task)
New payment gateways
New product types beyond the five existing ones
Steps
Blog seed script — Write a Node.js seed script that inserts 20 blog posts into the blog_posts table via Drizzle; each post has unique slug, category (Style Tips / Printing Guide / Gift Ideas / Corporate Wear / Care & Maintenance), 3–5 tags, cover image from Unsplash CDN URLs, and a realistic HTML body with H2/H3 headings, one FAQ section, and 600–800 words of Bangladesh-relevant apparel content
Run and verify seed — Execute the seed script and confirm all 20 posts appear in the admin blog list and public blog page with correct SEO meta rendered
3D model geometry upgrade — Regenerate hoodie, longsleeve, cap, and mug GLBs using the existing scripts/generate-models.cjs generator with higher segment counts and improved UV islands; re-export and replace files in public/models/
Fabric texture realism — Enhance garment3d.tsx procedural texture generator to produce a tighter, more photorealistic cotton-weave pattern with subtle normal-map-style bumpiness via canvas pixel manipulation; add a ceramic gloss sheen to the mug material using MeshPhysicalMaterial
3D loading & fallback polish — Ensure the Suspense loading skeleton matches the final render dimensions, add a subtle shimmer animation, and confirm the 2D canvas fallback never activates for WebGL2-capable browsers in the storefront
Spin wheel upgrade — Update prize definitions, sector colors, and label copy; add a subtle confetti burst on win using a lightweight canvas animation
Flash sale bar & exit-intent polish — Refine the animated gradient, update default copy to reference Bangladeshi seasonal sales, strengthen the exit-intent headline and layout for mobile

#20 - Full Website Audit, Fix & Feature Completion
What & Why
Comprehensive end-to-end audit and fix pass covering all outstanding critical bugs in the Design Studio, full site responsiveness, promotional popup visibility, 3D realism, and completion of all remaining pending features (blog sitemap images, sort-by-views, trending badge, mobile category reorder, category post-count badge, reassign-warning). All changes then pushed to GitHub.

Done looks like
Drop zone — tapping "Drop image here" on mobile opens the file picker; dragging a file on desktop works; the drop area has pointer-events: auto so clicking it works too
Back-side colour — switching to the Back face only tints the shirt shape; the card background stays warm-grey; the back cutout PNGs are regenerated with proper transparency if any are found to have solid white backgrounds
Spinner — every async action (upload, remove-bg, add-to-cart, 3D load) shows a loading indicator in-viewport; no scrolling needed; spinner dismisses correctly
SpinWheel / promos — SpinWheel appears on first visit and correctly resets so admins can preview it; AbandonedCartPopup z-index is raised so it never gets buried; ExitIntentPopup fires reliably on mobile scroll-up
Responsive layout — Design Studio, product pages, cart, checkout, blog, and admin all render cleanly on 360px–768px with no horizontal overflow or clipped buttons
Admin panel — all admin routes load error-free; CRUD with toast feedback; no broken API calls
3D realism — garment rendered with PBR fabric material (roughness ~0.88, subtle sheen), proper key + fill lighting, design texture crisp on mesh, no cross-face bleed
Blog sitemap images — <image:image> tags included for each blog post that has a featured image
Sort by most viewed — blog listing page has a "Most Viewed" sort option that orders posts by view count descending
Trending badge — posts with view count ≥ a configurable threshold show a 🔥 Trending badge on cards
Mobile touch reorder — category drag-to-reorder in the admin blog manager works with touch gestures on mobile
Category post-count badge — each category row in the manager shows the number of posts using it
Reassign-to-deleted warning — if admin selects a reassign target that is the same category being deleted, they see an error before the action proceeds
GitHub push — all changes committed with a descriptive message and pushed to georgelsmith333-hub/trynex-liestyle
Out of scope
New product types or new 3D garment models
Payment gateway changes
Google Search Console verification (handled by Task #13 which is already implemented)
Steps
Fix drop zone — Add onClick to the drop-zone container to trigger the hidden file-input on both desktop and mobile; set cursor: pointer and remove pointer-events-none from the inner UI; keep onDrop/onDragOver for desktop DnD.

Fix back-side colour flood — Inspect white-tshirt-back-cutout.png, white-hoodie-back-cutout.png, and white-longsleeve-back-cutout.png for non-transparent backgrounds; if any have white/solid backgrounds, use the built-in /api/remove-bg endpoint to regenerate them with alpha transparency and overwrite the files in public/mockups/; also add a CSS background: transparent and isolation: isolate on the SVG wrapper so no parent colour leaks in.

Fix spinner visibility — For each async flag (isAddingToCart, 3D Suspense, isRemoving, isUpscaling), ensure the spinner is rendered inside a fixed inset-0 overlay or as an inline spinner directly within the triggering button; remove any spinner that relies on the user scrolling to the bottom; raise AbandonedCartPopup z-index from z-50 to z-[500].

Fix SpinWheel display — Make the "Prize Wheel" admin settings toggle reset spin_modal_shown_v2 in localStorage when turned ON, so admins can preview it and returning users get a fresh prompt on their next visit after a configurable number of days; also add a data-testid to the wheel for easier debugging.

Responsive audit and fixes — On a 390px viewport, fix any horizontal overflow in DesignStudio.tsx (product tab scrollbar, colour swatches wrapping, right panel), ProductDetail.tsx, Cart.tsx, Checkout.tsx, Blog.tsx, and all admin pages; use overflow-x: auto snap, flex-wrap, or stacked card layouts consistent with the existing design language.

Admin panel health check — Load every admin route in sequence; fix any API calls missing auth headers; verify product create/edit, blog post publish, settings save, and category management all complete with correct toast notifications; fix any TypeScript errors surfaced by the audit.

3D garment realism — Upgrade MeshStandardMaterial to MeshPhysicalMaterial with roughness: 0.88, sheen: 0.3; add a subtle procedural canvas normal-map for cotton-weave texture; add a soft directional key light plus a low-intensity fill light; apply anisotropy: 4 on the design texture; ensure the UV projection doesn't bleed design onto the wrong hemisphere.

Blog sitemap images — Update the sitemap API route to include <image:image> XML tags for each blog post that has a coverImage; validate the output against the sitemap protocol.

Sort by most viewed — Add a sort=views query param to the blog API; on the Blog page add a "Most Viewed" chip/button to the existing sort controls; wire it to refetch with sort=views.

Trending badge — Add a trending boolean to the blog post query response (true when views >= threshold, where threshold defaults to 500 and is configurable in admin settings); render a 🔥 badge chip on blog cards and the post header.

Mobile touch reorder for categories — In the AdminBlog category manager, add touch event handlers (onTouchStart, onTouchMove, onTouchEnd) that mirror the existing mouse-drag reorder logic so admins can drag-sort on mobile.

Category post-count badge — In AdminBlog, add the post count (already computed client-side from the loaded posts list) as a small grey badge next to each category name in the category manager list.

Reassign-to-deleted-category warning — Before confirming a category deletion with a reassign target, check if the selected reassign target equals the category being deleted (case-insensitive); if so, block the submit and show an inline error: "Cannot reassign posts to the category you are deleting."

E2E check and GitHub push — Walk through design → cart → checkout, admin login → products → blog → settings; fix any remaining console errors; commit all changes and push to GitHub using the established token method.
